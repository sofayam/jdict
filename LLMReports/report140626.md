## 1. Wiki index тАФ Recent tab тЖТ Lists tab with sub-sections

The **Recent** tab on the wiki index page was renamed to **Lists** and
restructured to use sub-tab navigation instead of stacking all sections
vertically.

### Sub-tabs

Three sub-tabs are available within the Lists tab:

| Sub-tab     | Content |
|-------------|---------|
| **Changed** | Words sorted by `updated_at` DESC (previously "Recently changed") |
| **Added**   | Words sorted by `created_at` DESC (previously "Recently added") |
| **Podcast** | Words sorted by podcast-context count DESC (new) |

Only one sub-panel is visible at a time, sidestepping a rendering issue
where stacked sections could become invisible on some browsers.

### Podcast-linked query

New SQL in `wiki.js:getWikiBrowseData()` counts contexts with a
non-null `$.podcast` field:

```sql
SELECT w.slug, COUNT(*) as podcast_count
FROM wiki_words w, json_each(w.contexts) as ctx
WHERE json_extract(ctx.value, '$.podcast') IS NOT NULL
GROUP BY w.slug
ORDER BY podcast_count DESC
LIMIT 20
```

## 2. better-sqlite3 upgraded for Node 26

`npm rebuild` deleted the compiled native binary for `better-sqlite3` and
failed to recompile (Node 26 V8 API changes). Fixed by upgrading to the
latest version:

```
npm install better-sqlite3@latest
```

## 3. Dockerfile cleaned up

compile `better-sqlite3`'s native addon. Since the upgrade to the latest
`better-sqlite3` (which ships prebuilt binaries for `linux-x64`), these are
no longer needed. Removed the `RUN apk add` line entirely.
